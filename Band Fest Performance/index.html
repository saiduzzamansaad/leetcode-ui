Band Fest Performance Order System

Organizing performance slots for 8 prominent Bangladeshi bands using sorting algorithms

Problem Statement

A musical club of Leading University wants to organize a band fest with 8 prominent bands of Bangladesh:

WARFAZE
SHUNNO
ARTCELL
NEMESIS
AURTHOHIN
MEGHDOL
CHIRKUT
BLACK

The organizers need to determine the performance order considering two cases:

Case 1: First Letter Sorting

The band whose name's first letter comes first in the English alphabet will perform first, with subsequent slots filled by the next alphabetical bands.

Case 2: Same First Letter Resolution

When two bands share the same first letter, sort all letters of both names using a recursive technique. The band requiring more passes to sort will perform later.

Selection Sort Algorithm

Used for Case 1 to sort bands by their first letter:

  1. Iterate through all bands
  2. Find the band with the smallest first letter
  3. Swap it with the first position
  4. Repeat for remaining positions
function selectionSort(bands) {
    for (let i = 0; i < bands.length; i++) {
        let minIndex = i;
        for (let j = i + 1; j < bands.length; j++) {
            if (bands[j][0] < bands[minIndex][0]) {
                minIndex = j;
            }
        }
        [bands[i], bands[minIndex]] = [bands[minIndex], bands[i]];
    }
    return bands;
}

Time Complexity: O(n²) - Efficient for small datasets like our 8 bands

Recursive Bubble Sort

Used for Case 2 to sort individual band names:

  1. Compare adjacent letters in the name
  2. Swap if they're in the wrong order
  3. Repeat for the entire name
  4. Recursively sort the remaining letters
function recursiveBubbleSort(arr, n) {
    if (n === 1) return;
    for (let i = 0; i < n - 1; i++) {
        if (arr[i] > arr[i + 1]) {
            [arr[i], arr[i + 1]] = [arr[i + 1], arr[i]];
        }
    }
    recursiveBubbleSort(arr, n - 1);
}

Time Complexity: O(n²) - We track passes to determine sorting complexity

Case 2: Same First Letter Resolution

When bands share the same first letter, we use recursive bubble sort to determine performance order:

  • Sort all letters in each band name individually
  • Count the number of passes required to fully sort each name
  • The band requiring more passes to sort will perform later
  • This approach rewards bands with names that are "more sorted" naturally

Why This Approach Works

The number of passes needed in recursive bubble sort indicates how "unsorted" the original name was. More passes mean more inherent disorder, suggesting the band name is more complex and thus should perform later, while simpler, more alphabetically ordered names perform earlier.
