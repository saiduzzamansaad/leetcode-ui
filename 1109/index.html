Algorithm Explained

Brute Force Approach

// Simple but inefficient - O(n*m) complexity
function bruteForce(bookings, n) {
    const answer = new Array(n).fill(0);
    
    for (const [first, last, seats] of bookings) {
        for (let i = first; i <= last; i++) {
            answer[i-1] += seats;
        }
    }
    
    return answer;
}
  • For each booking, iterate through all flights in its range
  • Time complexity grows quickly with large inputs
  • Not suitable for production systems

Optimal Difference Array

// Efficient - O(n+m) complexity
function optimalSolution(bookings, n) {
    const answer = new Array(n).fill(0);
    
    // Mark range boundaries
    for (const [first, last, seats] of bookings) {
        answer[first-1] += seats;
        if (last < n) answer[last] -= seats;
    }
    
    // Calculate prefix sum
    for (let i = 1; i < n; i++) {
        answer[i] += answer[i-1];
    }
    
    return answer;
}
  • Marks where seat ranges begin and end
  • Calculates final counts with a single pass
  • Perfect for large datasets
  • Used in production systems

Visual Explanation

Key Concepts:

  1. Mark the start of each range with +seats
  2. Mark the end+1 of each range with -seats
  3. Calculate prefix sum to get final counts
  4. Works for overlapping ranges perfectly

The difference array technique is widely used in scheduling, calendar, and resource allocation systems.
